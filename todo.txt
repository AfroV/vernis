No — it won't pull from your new repo automatically. There are two separate GitHub configs:

/opt/vernis/github-config.json — For CSV collection sync (library content). Has owner, repo, path, token fields.

/opt/vernis/update-config.json — For code updates. Has github_repo and github_branch fields. Defaults to empty — no repo configured.

The github-update.sh script takes the repo as a CLI argument (sudo bash github-update.sh AfroV/vernis main), it doesn't auto-read from a config file.

To make your devices pull updates from the new repo, you'd need to either:

Set github_repo: "AfroV/vernis" in the update config on each device
Or since the repo is private, the devices would also need a GitHub token to access it
Right now hitting "update" on the control panel runs apt-get upgrade (system packages), not a GitHub pull. The GitHub update is a separate script.
